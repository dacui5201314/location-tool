## 二、第二阶段目标

### 2.1 报告精准度提升（P0）

**问题**: 链锁加盟品牌未正确识别为直接竞品，导致报告竞品分析不准确。

**方案**:
1. 导出生产 AnalysisRecord 数据 → 本地分析 POI 分类漏洞
2. 按品牌名称频率检测链锁品牌（同名≥3即疑似连锁）
3. 补充 `industry_config.py` 品牌名单和排除规则
4. 增量更新 `check_industry_rigor_rules.py` 测试用例
5. 每次修改后跑全量测试验证

**预计工作量**: 2-3天
**负责人**: Claude + Codex 联合

---

### 2.2 报告分享与传播（P1）

**2.2.1 报告 H5 分享链接**
- 生成可分享的加密链接（含 token），微信内打开可查看完整报告
- 支持分享到微信好友、群聊、朋友圈
- 链接有效期 7 天，过期失效

**2.2.2 邮件发送报告**
- 用户输入邮箱 → 服务器生成 HTML 报告 PDF → 邮件发送
- 后端新增 `POST /api/user/report/email` 接口
- 使用 SMTP 发送，配置在管理后台

**预计工作量**: 2天

---

### 2.3 会员体系完善（P1）

**2.3.1 积分/等级系统**
- 新增积分获取途径：每日签到、分享报告、邀请好友
- 会员等级差异化权益：免费报告数/月、并发分析数、报告保存期

**2.3.2 优惠券/折扣**
- 管理后台生成折扣码（百分比/固定金额）
- 购买套餐时自动计算折后价

**预计工作量**: 3-4天

---

### 2.4 运营工具（P2）

**2.4.1 运营数据看板增强**
- 日报/周报自动生成
- 用户留存分析
- 业态热度排名
- 地区分析量热力图

**2.4.2 小程序订阅消息**
- 报告完成通知
- 会员到期提醒
- 新功能/优惠推送

**预计工作量**: 3-4天

### 2.5 用户体验优化（P2）

**2.5.1 多门店对比分析**

**预计工作量**: 2-3天
