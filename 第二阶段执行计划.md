## 二、第二阶段目标

### 2.1 报告精准度提升（P0）

**问题**: 链锁加盟品牌未正确识别为直接竞品，导致报告竞品分析不准确。

**核心思路**: 用生产数据驱动规则迭代。服务器已积累大量 AnalysisRecord，从中提取 POI 频次/分类偏差，反向补充 industry_config.py 和 amap_service.py。

#### 2.1.1 生产数据采集与分析
- 从 `AnalysisRecord` 表导出最近 N 天的报告数据（report_json + real_data + fact_errors）
- 统计高频 POI 名称，识别疑似链锁品牌（同名出现≥3次且不在现有 name_keywords 中）
- 统计 fact_errors 触发最频繁的业态/维度，优先修复
- 输出一份《POI 分类偏差报告》供 Codex 审计

#### 2.1.2 品牌识别规则增强
- 在 `industry_config.py` 中新增品牌识别辅助字段（如 `brand_suffixes`、`franchise_keywords`）
- 补充常见加盟品牌（串串、麻辣烫、炸鸡、奶茶、卤味等）到直接竞品 name_keywords
- 在 `amap_service.py` 的 `classify_poi_rigor()` 中增加品牌名模糊匹配层

#### 2.1.3 子业态精细化
- 审核当前 43 个前台入口 → 14 个 Master 集群的映射关系
- 识别缺失子业态（如"轻食沙拉""精酿啤酒""宠物医院"等新兴品类）
- 补充 subtype 规则，确保子业态不会被错误归类到"无关项"

#### 2.1.4 测试体系加固
- 每个品牌/子业态修改后，同步增补 `check_industry_rigor_rules.py` 测试用例
- 目标：从当前 2178 PASS 提升到 2500+ PASS
- 新增"品牌识别专项测试"模块，覆盖已识别链锁品牌

#### 2.1.5 LLM 幻觉进一步压制
- 审查近期 fact_errors 中的 P0-NAME 命中模式
- 调整 `location_analysis.py` prompt 中品牌引用的约束措辞
- 考虑在 prompt 中显式注入"已知设施白名单"减少幻觉

**预计工作量**: 3-4天
**负责人**: Codex（审计+方案）+ Claude（执行+验证）
