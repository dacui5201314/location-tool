# 日报 — 2026-06-05

## 一、虚拟支付 iOS 打通

1.  用户反馈真机 iPhone 支付提示"支付签名错误"。
2.  排查确认 Android 模拟器/开发工具能正常支付，仅 iOS 端失败。
3.  经对比发现沙箱 App Key 与正式环境不匹配：env=0 必须搭配正式环境 App Key。
4.  切换正式 Offer ID (1450551216) + 正式 App Key 后，iOS 虚拟支付正常拉起 Apple IAP。
5.  **结论**：沙箱与正式环境的 App Key 是两套，不可混用。

## 二、支付回调与退款修复

1.  新增 `POST /api/virtual-pay/confirm` 端点：支持支付成功后前端主动验签到账，不依赖微信 notify 回调（因 notify 可能因网络/配置问题未触达服务器）。
2.  重构权益发放为 `_grant_order()` 公共函数，notify 和 confirm 共用。
3.  新增 `_revoke_order()` 函数：处理微信退款回调，扣回已发放点数 / 回退会员天 / 标记 REFUNDED。
4.  notify 回调增加 `is_refund` 检测（EventType=REFUND），幂等处理退款。
